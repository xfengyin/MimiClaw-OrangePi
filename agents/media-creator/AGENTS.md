## 内容规范

### 写作风格

- **技术准确**: 确保代码和术语正确
- **通俗易懂**: 把复杂技术讲得简单
- **结构清晰**: 使用列表和标题
- **图文并茂**: 适当添加图表

### 标题格式

```markdown
## 大标题 (页面标题)
### 二级标题 (章节)
#### 三级标题 (小节)
```

### 代码块

````markdown
```c
// C 代码示例
int example(void) {
    return 0;
}
```
````

## 版本发布流程

### 发布前

1. 从 Dev-Planner 获取发布内容
2. 确认 CHANGELOG 更新
3. 检查版本号一致性

### 发布中

```bash
# 创建 GitHub Release
git tag v2.0.1
git push origin v2.0.1
# GitHub Actions 自动创建 Release
```

### 发布后

1. 社交媒体宣传
2. 回答用户问题
3. 收集反馈

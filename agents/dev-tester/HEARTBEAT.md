# ============================================================================
# Dev-Tester 心跳任务
# ============================================================================

## 定时检查 (每 15 分钟)

### 1. 测试执行检查
- 运行 `make -C libs test`
- 运行 `make -C tests`
- 记录测试结果和失败用例

### 2. 覆盖率监控
- 检查覆盖率报告
- 对比上次覆盖率
- 低于 80% 标记为问题

### 3. Bug 追踪检查
- 检查新增 Bug 报告
- 验证修复是否有效
- 更新 Bug 状态

### 4. 性能基准检查
- 运行 `cd libs/tests/benchmarks && ./bench_*`
- 对比性能目标
- 记录异常

## 单元测试检查
- [ ] make -C libs test 全部通过
- [ ] 覆盖率 > 80%
- [ ] 无阻塞级 Bug

## 集成测试检查
- [ ] make -C tests 全部通过
- [ ] 核心流程验证通过

## 性能测试检查
- [ ] 启动时间 < 1s
- [ ] 响应延迟 < 200ms
- [ ] 内存占用 < 50MB

## Bug 状态检查
- [ ] 阻塞 Bug: 0
- [ ] 高优先级: 待修复
- [ ] 中/低优先级: 记录跟踪

- [ ] 每日生成测试报告

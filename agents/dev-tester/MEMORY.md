# 测试相关信息

## 测试架构

### 测试目录结构
```
tests/
├── unit/           # 单元测试
│   ├── core/
│   ├── memory/
│   ├── config/
│   ├── plugins/
│   └── tools/
├── integration/    # 集成测试
└── benchmarks/     # 性能测试
    ├── bench_memory.c
    ├── bench_latency.c
    ├── bench_startup.c
    └── test_concurrent.c
```

### 测试框架
- **框架:** cmocka
- **编译:** CMake
- **覆盖率:** lcov + genhtml

## 测试命令

```bash
# 单元测试
make -C libs test

# 集成测试
make -C tests

# 覆盖率报告
make COVERAGE=1
make -C libs test
# 查看 coverage-report/

# 性能基准测试
cd libs/tests/benchmarks
./bench_memory      # 内存性能
./bench_latency     # 延迟测试  
./bench_startup     # 启动时间
```

## 质量标准

### 测试通过条件
| 指标 | 要求 | 优先级 |
|------|------|--------|
| 单元测试 | 全部通过 | 必须 |
| 覆盖率 | > 80% | 必须 |
| 阻塞 Bug | 0 | 必须 |
| 高优先级 Bug | < 3 | 重要 |
| 性能延迟 | < 200ms | 重要 |
| 内存占用 | < 50MB | 重要 |

### Bug 严重程度定义

| 级别 | 描述 | 影响 |
|------|------|------|
| **阻塞** | 系统崩溃、数据丢失 | 停止测试 |
| **高** | 核心功能不可用 | 下版本修复 |
| **中** | 功能异常可绕过 | 记录跟踪 |
| **低** | 界面/体验问题 | 后续优化 |
